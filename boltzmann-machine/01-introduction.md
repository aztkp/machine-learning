# 1. 導入と歴史

## ボルツマンマシンとは

**ボルツマンマシン（Boltzmann Machine）** は、1985年にジェフリー・ヒントンとテリー・セジュノスキーによって提案された**確率的生成モデル**です。

統計力学の概念を人工ニューラルネットワークに応用したもので、名前は物理学者ルートヴィヒ・ボルツマンに由来します。

## なぜ「ボルツマン」なのか

ボルツマンマシンは、**ボルツマン分布**（統計力学における基本的な確率分布）に基づいています。

システムがエネルギー $E$ を持つ状態にある確率は：

$$P(E) = \frac{e^{-E/kT}}{Z}$$

- $k$: ボルツマン定数
- $T$: 温度
- $Z$: 分配関数（正規化定数）

この考え方をニューラルネットワークに適用したのがボルツマンマシンです。

## 歴史的背景

### 1980年代：誕生

```
1982年 - ホップフィールドネットワーク登場
         （決定論的なエネルギーベースモデル）

1985年 - ボルツマンマシン提案
         （確率的な拡張）
```

ホップフィールドネットワークは局所最小値に陥りやすい問題がありました。ボルツマンマシンは**確率的な状態遷移**を導入することで、この問題を緩和しました。

### 1990年代：冬の時代

- 学習が非常に遅い
- 計算コストが高い
- バックプロパゲーションの隆盛

### 2000年代：復活

```
2002年 - コントラスティブダイバージェンス（CD）アルゴリズム
         → 学習が劇的に高速化

2006年 - 深層信念ネットワーク（DBN）
         → ディープラーニング革命の口火
```

## ボルツマンマシンの位置づけ

```
ニューラルネットワーク
├── フィードフォワードネット（MLP, CNN）
├── リカレントネット（RNN, LSTM）
└── エネルギーベースモデル ← ここ
    ├── ホップフィールドネットワーク（決定論的）
    └── ボルツマンマシン（確率的）
        ├── 完全ボルツマンマシン
        ├── 制限ボルツマンマシン（RBM）
        └── 深層ボルツマンマシン（DBM）
```

## 主な特徴

| 特徴 | 説明 |
|------|------|
| 確率的 | ノードの状態が確率的に決まる |
| 無向グラフ | 接続に方向がない |
| エネルギーベース | システム全体のエネルギーで定義 |
| 生成モデル | データの分布を学習して生成できる |

## 現代での意義

現在、ボルツマンマシン自体が実用で使われることは少なくなりましたが、その概念は以下に影響を与えています：

1. **生成モデルの基礎理論**
   - VAE、GANなどの理論的基盤

2. **エネルギーベース学習**
   - 最新の研究でも活発に使われる概念

3. **量子コンピューティング**
   - 量子アニーリングとの関連
